Reglas generales para TODAS las capturas (para que el manual sea “a prueba de novatos")
Cada captura debe incluir título, flechas/resaltado, y 2–4 líneas de explicación.
En capturas de código: que se vea nombre del archivo, y si pueden, líneas o al menos el bloque completo.
Siempre que haya un “id” o un nombre técnico, deben mostrar dónde se define y dónde se usa (dos capturas enlazadas).
Si un paso depende de otro, deben poner un mini “Checklist” debajo: “Antes de esto, verifica X”.


El manual debe permitir que cualquier persona pueda configurar correctamente el módulo usando únicamente ese documento. Para ello, es imprescindible sea muy visual. La idea es que el manual sirva como guía completa de setup del modelo, paso a paso, para una persona que lo hace por primera vez.
